## Decisions Made
- **buttonVariants() over Button asChild:** The plan's action text instructed wrapping CTAs in `<Button asChild><Link>...</Link></Button>` per the standard Radix-based shadcn pattern. This codebase's `Button` component (`src/components/ui/button.tsx`) wraps `@base-ui/react/button`, which has no `asChild` prop — `tsc --noEmit` caught this immediately (`Property 'asChild' does not exist on type ...`). Used the already-exported `buttonVariants()` CVA helper applied directly to the `Link`'s `className` instead — this was explicitly anticipated as a fallback option in both RESEARCH.md ("use shadcn's own `buttonVariants({ variant: "default" })` className helper, or wrap in `<Button asChild>...`") and PATTERNS.md, so no scope deviation, just selecting the correct one of two pre-documented options for this codebase's actual Button implementation.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `Button asChild` does not compile against this codebase's Button component**
- **Found during:** Task 2 (owner dashboard CTA conversion)
- **Issue:** Plan instructed `<Button asChild><Link href="/equipment/new">List equipment</Link></Button>`. `npx tsc --noEmit` failed with `TS2322: Property 'asChild' does not exist on type ... ButtonProps`. Root cause: this project's `Button` (`src/components/ui/button.tsx`) is built on `@base-ui/react/button`, not Radix's `Slot`-based primitive that shadcn's standard `Button` uses — `asChild` is a Radix-Slot-only prop and was never implemented here.
- **Fix:** Replaced `<Button asChild>` usage with the exported `buttonVariants()` CVA helper applied directly to the `Link`'s `className` prop: `<Link href="..." className={buttonVariants()}>...</Link>`. Applied identically in both Task 2 (owner dashboard) and Task 3 (farmer dashboard) for consistency.
- **Files modified:** `src/app/(owner)/owner/dashboard/page.tsx`, `src/app/(farmer)/farmer/dashboard/page.tsx`
- **Verification:** `npx tsc --noEmit` returns clean (zero errors) after the fix; `grep -c "bg-slate-950"` returns 0 in both files.
- **Committed in:** `94e6023` (Task 2), `f644334` (Task 3)
