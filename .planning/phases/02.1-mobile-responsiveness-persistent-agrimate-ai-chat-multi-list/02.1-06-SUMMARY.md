## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Added `nativeButton={false}` to every `SheetClose` instance**
- **Found during:** Task 3 (human-verify checkpoint — live browser testing surfaced a console/runtime error before approval could be given)
- **Issue:** `@base-ui/react/dialog`'s `Close` primitive defaults `nativeButton={true}`, which assumes the rendered child is a real `<button>`. Wrapping a `Link` (anchor) without this flag throws "A component that acts as a button expected a native `<button>`..." as a dev-mode console error, escalating to an uncaught runtime error on subsequent renders.
- **Fix:** Added `nativeButton={false}` to all 6 `SheetClose` instances across both layout files (4 in farmer, 2 in owner).
- **Files modified:** `src/app/(farmer)/layout.tsx`, `src/app/(owner)/layout.tsx`
- **Verification:** `npx tsc --noEmit` clean; live-verified in browser at 375px for both roles post-fix — no console errors, drawer closes correctly on every nav link tap
- **Committed in:** `88ea3f6`
