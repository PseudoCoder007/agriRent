# Phase 02.1 Plan 03: Dark Mode Toggle Summary

Wired the already-installed-but-unused `next-themes` dependency into a working binary Sun/Moon light/dark toggle, persisted via next-themes' own localStorage mechanism, defaulting to `system` theme.

## What Was Built

1. **`src/components/theme/theme-provider.tsx`** — `"use client"` wrapper re-exporting `next-themes`' `ThemeProvider` (the exact Shadcn-documented pattern), forwarding all props including `children`.
2. **`src/components/theme/theme-toggle.tsx`** — Binary Sun/Moon icon button calling `useTheme().setTheme(theme === "dark" ? "light" : "dark")`. Sized `h-11 w-11 sm:h-9 sm:w-9` to satisfy the UI-SPEC's 44px mobile touch-target floor while collapsing to the standard icon-button size at `sm:` and above.
3. **`src/app/layout.tsx`** — Added `suppressHydrationWarning` to `<html lang="en">` (the only place it goes, per RESEARCH.md Pitfall 2). Wrapped `{children}` and `<Toaster />` in `<ThemeProvider attribute="class" defaultTheme="system" enableSystem disableTransitionOnChange>`. `attribute="class"` is mandatory because `globals.css`'s `@custom-variant dark (&:is(.dark *));` only reacts to a `.dark` class, not `data-theme`.
4. **`src/app/(farmer)/layout.tsx`** and **`src/app/(owner)/layout.tsx`** — Inserted `<ThemeToggle />` as the first child of the header's right-hand `<div className="flex items-center gap-2">`, before `<NotificationBell />`. Final control order: `ThemeToggle, NotificationBell, LogoutButton`. Auth/role-check logic (lines 19-38 in both files) was read but left completely untouched.
