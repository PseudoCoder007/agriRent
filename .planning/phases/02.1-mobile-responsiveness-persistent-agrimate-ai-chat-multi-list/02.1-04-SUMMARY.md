## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `SheetTrigger asChild` does not exist on this project's base-ui-backed primitives — used `render` prop instead**
- **Found during:** Task 2 (owner layout), before commit
- **Issue:** The plan's action text specified `<SheetTrigger asChild><Button ...>...</Button></SheetTrigger>`, following the conventional Radix-Slot-based shadcn pattern. This project's shadcn preset generates components backed by `@base-ui/react` (confirmed: `src/components/ui/button.tsx` imports `Button as ButtonPrimitive from "@base-ui/react/button"`; `sheet.tsx`'s generated `DialogTrigger` type — `SheetPrimitive.Trigger.Props` from `@base-ui/react/dialog` — has no `asChild` prop, only a `render` prop, confirmed by reading `node_modules/@base-ui/react/dialog/trigger/DialogTrigger.d.ts`). Writing `asChild` as specified would have been silently dropped as an unrecognized prop, producing an unstyled default `<button>` trigger instead of the intended ghost icon button.
- **Fix:** Used the `render` prop pattern already demonstrated inside the generated `sheet.tsx` itself (`SheetClose`'s `render={<Button variant="ghost" .../>}` usage) — `<SheetTrigger render={<Button variant="ghost" size="icon" className="h-11 w-11 sm:hidden" aria-label="Open navigation menu" />}><Menu className="h-5 w-5" /></SheetTrigger>`. This produces the identical visual/functional result (Button-styled trigger, Menu icon as children) without relying on a prop that doesn't exist on this codebase's primitives. Matches the precedent already noted in STATE.md decisions: "[Phase 02.1-02]: Used buttonVariants() className helper instead of Button asChild for Link-styled CTAs -- this codebase's Button wraps @base-ui/react/button (no asChild prop)."
- **Files modified:** `src/app/(owner)/layout.tsx`, `src/app/(farmer)/layout.tsx`
- **Commit:** dd5c54d (owner), dbe044b (farmer)
