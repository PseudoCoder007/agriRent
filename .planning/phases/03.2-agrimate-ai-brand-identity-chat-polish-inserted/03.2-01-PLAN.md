<objective>
Most of Phase 3.2's scope already shipped, built in parallel outside this plan's execution: the `AgriMateAIIcon` shared brand-mark component (navbar/avatar/favicon variants), the rebranded chat widget header ("Chat with AgriMate AI"), the branded per-message assistant avatar, the animated `TypingDots` typing indicator, and a 64x64 `ImageResponse`-generated favicon (`src/app/icon.tsx`). This plan documents that existing work as the retroactive majority of the phase, and implements the one confirmed remaining gap: `src/app/apple-icon.tsx` currently just re-exports `icon.tsx`'s 64x64 output unchanged (`export { contentType, default, size } from "./icon"`), which renders soft/blurry on iOS home screens where Apple touch icons are expected at 180x180. This plan gives `apple-icon.tsx` its own 180x180 `size` export and its own `ImageResponse` render at that resolution.

Purpose: Per BRAND-01, the favicon and Apple touch icon must each render the brand mark "at a correct, non-blurry size for their respective surfaces (browser tab vs. iOS home screen)" — confirmed via Next.js's own `next/og` documentation that the `size` export is passed directly into `ImageResponse`'s render dimensions (the official example spreads `...size` into `ImageResponse`'s options, and the SVG inside is sized via `width: '100%', height: '100%'` to fill that canvas), so a dedicated 180x180 `size` + matching `<svg width={180} height={180} viewBox="0 0 64 64">` produces a sharp, non-interpolated render rather than browser-upscaling a 64x64 raster.

Output: `src/app/apple-icon.tsx` rewritten with its own `size = { width: 180, height: 180 }` export and a standalone `ImageResponse` call rendering the same leaf-mark SVG paths (copied from `icon.tsx`, same `viewBox="0 0 64 64"` coordinate system) at 180x180 pixel output. All other Phase 3.2 artifacts (brand mark component, chat widget, favicon route) are confirmed already complete and are explicitly NOT modified by this plan.
</objective>
