# Plan 05-01: AI recommendations and safe tool-calling

## Goal

Add an AI layer that can recommend equipment and optionally assist discovery using safe, validated calls into existing marketplace services.

## Deliverables

- Recommendation-focused AI behavior built on existing booking/review/listing data
- A safe tool-calling boundary, if used, that only reaches approved service helpers
- A user-facing way to surface recommendations or suggestions without making the AI autonomous

## Constraints

- No direct database access from the AI layer
- No writes initiated by AI
- Human confirmation remains required for any marketplace-changing action

## Verification

- Manual checks that AI output is grounded in app data
- Manual checks that no unsafe tool path exists
- Typecheck remains green
