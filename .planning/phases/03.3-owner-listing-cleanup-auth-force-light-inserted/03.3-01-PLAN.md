# Phase 03.3 Plan 01: Owner Listing Cleanup and Forced-Light Auth Fixes

## Objective

Fix the three owner/auth regressions reported on 2026-06-28:

1. Deleted owner listings still appearing after refresh.
2. No persistent way to add another equipment listing from the owner dashboard.
3. Login/signup pages rendering dark after logout or theme toggles.

## Implementation Changes

- Update the owner equipment read path so the dashboard only receives active listings.
- Keep public browse/detail behavior unchanged: deleted listings should remain unavailable there.
- Add a persistent `List equipment` action to the owner dashboard header or top action area so it is always visible.
- Apply the forced-light page contract to the auth pages and the landing page so logout does not inherit dark state.
- Add and maintain one debug note per bug under `.planning/debug/open/`, then move each to `resolved` after verification.

## Bug Tracking

- `owner-deleted-listing-still-visible-after-refresh`
- `owner-dashboard-missing-persistent-list-equipment-cta`
- `auth-pages-black-after-dark-mode-logout`

## Test Plan

- `cmd /c node_modules\.bin\tsc.cmd --noEmit`
- Manually delete a listing, refresh the owner dashboard, and confirm the item disappears.
- Manually confirm the owner dashboard still offers `List equipment` after the delete.
- Manually click a deleted listing URL and confirm it goes to not-found.
- Toggle dark mode, log out, and confirm login/signup/forgot/reset/home render light again.

## Assumptions

- No schema migration is needed.
- Soft-deleted listings should be hidden from the owner dashboard entirely.
- The persistent create CTA should remain visible regardless of how many active listings exist.
- The forced-light behavior on auth/home pages is intentional and should stay explicit.
