# Phase 03.3 Research

## Summary

This phase targets three user-visible regressions confirmed in the current checkout:

1. An owner deletes a listing, refreshes, and still sees it on the dashboard.
2. The owner dashboard does not always show a persistent way to create another listing.
3. After dark mode is toggled and the user logs out, the auth pages can remain dark/black instead of returning to the intended light presentation.

The service layer and route structure already support multi-listing owners. The issue is the owner dashboard read/render path and the page-level theme behavior, not a schema limitation.

## Findings

- `src/lib/services/listing.service.ts` currently returns owner equipment without filtering `deleted_at` in `getEquipmentByOwner()`.
- `src/app/(owner)/owner/dashboard/page.tsx` only exposes `List equipment` inside the empty state, so an owner with one or more items has no persistent create CTA.
- The auth pages are expected to remain light, but the current behavior after logout shows that the page wrappers are still inheriting dark theme state somewhere in the page shell.

## Decision

- Treat deleted equipment as hidden from the owner dashboard read path.
- Add a persistent dashboard create CTA, not just an empty-state CTA.
- Make the auth/home pages explicitly force-light so logout does not depend on the previous theme.

## Bug Tracking

- Create open debug notes for the three issues in `.planning/debug/open/`.
- Close each note only after the corresponding fix is verified and the behavior is observed in the browser.
