# Plan 03-03: Dashboard enrichment

## Goal

Turn both dashboards into informative, summary-driven views that surface bookings, reviews, and notification activity.

## Deliverables

- Richer owner dashboard metrics and recent activity sections
- Richer farmer dashboard metrics and recent activity sections
- No new notification route; keep the existing dashboard embedding model

## Constraints

- Keep the existing role-specific route structure
- Prefer computed summaries from existing services over stored denormalized counters
- Preserve the current booking and notification action flows

## Verification

- Manual check that both dashboards still load correctly for their roles
- Confirm the new summaries reflect real data, not placeholders
