# Plan 03-02: Review service and detail page flow

## Goal

Let a farmer submit a review only after a completed booking, and show average rating plus review list on the equipment detail page.

## Deliverables

- `src/lib/services/review.service.ts`
- `src/lib/validations/review.schema.ts`
- `src/app/actions/review.actions.ts`
- Review display and submission wiring in `src/app/(farmer)/equipment/[id]/page.tsx`

## Constraints

- Service layer owns all business rules and duplicate detection
- Review form only renders when the server says the current user is eligible
- Do not allow editing or deleting reviews in this phase

## Verification

- Add Vitest coverage for review creation and read helpers
- Manually confirm pending/approved/cancelled bookings are rejected
- Confirm completed-booking reviews appear on the equipment detail page
