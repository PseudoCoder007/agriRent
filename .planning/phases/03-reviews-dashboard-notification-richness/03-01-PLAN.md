# Plan 03-01: Reviews schema foundation

## Goal

Add the database foundation for trustworthy reviews and, if needed, make notification rows available to Realtime publication in an idempotent way.

## Deliverables

- New migration file under `supabase/migrations/` for the reviews table
- `reviews` table with booking/equipment/farmer linkage, rating, comment, timestamps, and one-review-per-booking protection
- RLS policy set that only allows a farmer to insert a review for a completed booking they own
- Any minimal publication adjustment needed so live notifications can be subscribed to on the client

## Constraints

- Do not alter application code in this plan
- Do not change existing booking or notification semantics
- Keep the review authorization check server-side and avoid recursive RLS lookups

## Verification

- Typecheck remains green after the migration file is added
- Phase 3 review service can later rely on the schema without additional schema edits
