# Plan 03-04: Live notifications

## Goal

Replace the plain notification list with a live bell and unread count that updates through Supabase Realtime.

## Deliverables

- Shared `NotificationBell` client component
- Realtime subscription for notification inserts/updates
- Mark-read action so unread counts decrease without a page reload
- Header/layout integration for both owner and farmer routes

## Constraints

- Keep the client subscription lightweight
- The unread count should come from the `read` field, not a separate table
- No websocket server or background worker

## Verification

- Manual check that a newly inserted notification appears live
- Manual check that marking a notification read updates the bell count
