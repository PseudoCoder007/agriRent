## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Chat widget input -> /api/chat | Untrusted free-text user input crosses into an LLM completion request — must not be interpretable as instructions to perform DB writes |
| /api/chat -> NVIDIA NIM | Server-to-external-API boundary; API key must never cross back into client-visible code or responses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-------------------|
| T-04-01 | Information Disclosure | `NVIDIA_API_KEY` | mitigate | Key read only inside `src/lib/services/ai.service.ts` via `process.env.NVIDIA_API_KEY` (no `NEXT_PUBLIC_` prefix), file is server-only and never imported by `chat-widget.tsx` (a Client Component) — the only network boundary the browser ever talks to is `/api/chat`, never NVIDIA directly |
| T-04-02 | Elevation of Privilege / Tampering | AI layer as a backdoor to `bookings`/`equipments` | mitigate | `ai.service.ts` imports no Supabase client at all — no DB access path exists from the AI layer in Phase 1; no tool-calling, no function-calling schema referencing booking/listing services, per ARCHITECTURE.md Anti-Pattern 4 and REQ AI-02 |
| T-04-03 | Denial of Service | NVIDIA NIM 429 rate limit exhaustion (free tier ~40 RPM) | mitigate | Bounded retry (max 2 retries, exponential backoff) in `ai.service.ts`; client-side send-button disable while a request is in flight prevents rapid repeated submits compounding rate-limit pressure |
| T-04-04 | Information Disclosure | Raw provider error text relayed to client | mitigate | `getChatCompletion` returns a generic friendly fallback message on failure, never the raw NVIDIA error body or any string containing the API key |

</threat_model>
