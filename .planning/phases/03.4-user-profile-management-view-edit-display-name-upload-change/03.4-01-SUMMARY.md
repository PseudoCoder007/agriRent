## Decisions Made
- Documented the `phone` column's information-disclosure tradeoff directly in the migration SQL comment (mirrors the existing accepted `email` exposure) rather than silently extending it
- Did not gate avatar upload behind `public.is_owner()` — both farmers and owners need profile avatars
- Kept `avatarFileSchema` duplicated rather than cross-imported from `equipment.schema.ts` to keep the profile validation module self-contained
